Step 1: Fault Detection and Localization

1A

Connection Loss Detection

In case of fiber cable break, the connection between network devices is completely lost. This situation can be immediately detected with ping tests.

# Ping test during cable break PC-A> ping 192.168.1.11 Request timed out. Request timed out.
1B

Check Switch Interface Status

Check the status of relevant interfaces on switches to confirm the physical layer problem.

Switch# show interface GigabitEthernet0/1 GigabitEthernet0/1 is down, line protocol is down # This output indicates a physical connection problem

Step 2: Physical Repair Process

2A

Restoring Fiber Connection

While fusion splicing is done in real life, re-establishing the connection between switches in simulation is sufficient. This simulates physical layer repair.

2B

Verify Connection Test

After repair, verify that switch interfaces are in "up" state and ping tests are successful.

Switch# show interface GigabitEthernet0/1 GigabitEthernet0/1 is up, line protocol is up PC-A> ping 192.168.1.11 Reply from 192.168.1.11: bytes=32 time=1ms TTL=128 Reply from 192.168.1.11: bytes=32 time=1ms TTL=128

Step 3: Verification and Testing

3A

End-to-End Connection Test

After repair completion, comprehensive tests are performed to verify that communication between network devices is fully restored.

3B

Service Continuity Check

Ensure all network services are functioning normally. Test that critical applications are accessible.
