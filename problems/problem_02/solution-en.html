* Problem: PC4 was initially assigned to VLAN 20 (error). Moved to VLAN 10 in solution.

VLAN 10 (SALES): PC0, PC2, PC4 - 192.168.10.0/24 subnet

VLAN 20 (ENGINEERING): PC1, PC3, PC5 - 192.168.20.0/24 subnet

Trunk Ports: Carry VLAN 10 and 20 between switches

Step 1: Detecting Faulty Situation (Troubleshooting)

1A

Ping Test Between PCs

First, to confirm the existence of the problem, a ping is sent from PC0 (192.168.10.11) to PC4 (192.168.10.13) which should be in the same VLAN. The test is seen to fail.

PC0> ping 192.168.10.13 Pinging 192.168.10.13 with 32 bytes of data: Request timed out. Request timed out. Request timed out. Request timed out. Ping statistics for 192.168.10.13: Packets: Sent = 4, Received = 0, Lost = 4 (100% loss), # PC4 is in different VLAN, so ping fails
1B

Checking VLAN Status on Switches

Connect to Switch2 where the problem occurs and use the show vlan brief command to check which VLANs the ports are assigned to. The output of this command shows that PC4's connected FastEthernet0/1 port is assigned to VLAN 20 instead of the required VLAN 10.

Switch2# show vlan brief # Output shows that Fa0/1 port is in the wrong VLAN. VLAN Name Status Ports ---- ----------- -------- -------------------------------- 10 SALES active Fa0/2 20 ENGINEERING active Fa0/1, Fa0/4
1C

Checking Trunk Ports

Check the trunk ports that provide inter-switch connectivity using the show interfaces trunk command. In this scenario, it is understood that the trunk ports are properly configured and the problem does not originate from here.

Step 2: Fixing Configuration

2A

Assigning Access Port to Correct VLAN

The detected faulty port (FastEthernet0/1) is assigned to the correct VLAN 10 (SALES).

Switch2(config)# interface FastEthernet0/1 Switch2(config-if)# switchport access vlan 10 # This command assigns the port to the correct VLAN.

Step 3: Verification and Final Test

3A

Checking Configuration Again

After the correction, when the show vlan brief command is run again, it is seen that the FastEthernet0/1 port is now correctly listed under VLAN 10.

3B

Final Ping Test

After the correction is verified, a ping is sent again from PC0 to PC4. This time the test is seen to be successful and the problem is resolved.

PC0> ping 192.168.10.13 Pinging 192.168.10.13 with 32 bytes of data: Reply from 192.168.10.13: bytes=32 time<1ms TTL=128 Reply from 192.168.10.13: bytes=32 time<1ms TTL=128 Reply from 192.168.10.13: bytes=32 time<1ms TTL=128 Reply from 192.168.10.13: bytes=32 time<1ms TTL=128 Ping statistics for 192.168.10.13: Packets: Sent = 4, Received = 4, Lost = 0 (0% loss), Approximate round trip times in milli-seconds: Minimum = 0ms, Maximum = 0ms, Average = 0ms # âœ… PC4 is now in correct VLAN, ping successful!
3C

Saving Solution File

After all errors are corrected in Packet Tracer, the file is saved as vlan_solution.pkt.
